# Error `#1`: Active `ViewModel` not found

This happened because `vmLookup` for [`useViewModel`](/react/api/use-view-model) (React) / [`useViewModel`](/solid/api/use-view-model) (Solid) is not provided, and the hook tries to look up the active view model using `ActiveViewModelContext`, which exists only when using [`withViewModel`](/react/api/with-view-model) / [`withViewModel`](/solid/api/with-view-model).

## Explanation

This usage:

```tsx
const model = useViewModel<YourVM>();
```

uses `ActiveViewModelContext`, which exists only under a `withViewModel` HOC.

This can also happen if you access an active `ViewModel` that is not an ancestor in the current render tree:

```tsx
<Component1>
  <ComponentAChild1>
    <Component>
      {/* useViewModel<YourVM>() here — no active parent */}
    </Component>
  </ComponentAChild1>
  <YourVMComponent /> {/* provides active ViewModel via withViewModel */}
</Component1>
```

## Potential solution

Pass a [`vmLookup`](/api/other/view-model-lookup) to `useViewModel`:

```tsx
const model = useViewModel(YourVM);
const model = useViewModel("idofyourvm");
```

Or call `useViewModel` under the correct active `ViewModel` in the tree:

```tsx
<YourVMComponent> {/* withViewModel */}
  <Component1>
    <ComponentAChild1>
      <Component>
        {/* useViewModel<YourVM>() — OK */}
      </Component>
    </ComponentAChild1>
  </Component1>
</YourVMComponent>
```
